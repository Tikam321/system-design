 ###  functinoal requirement
- takes a long URL and generates a shorter alias" → CREATE operation (URL Shortening)
- redirects users to the original URL" → READ operation (URL Redirection)
- track link usage" → UPDATE/INCREMENT operation (Analytics)

  ###  non functional requirements
  - high availablity: make sure all the urls are accessible 24/7 .
  - low latanecy : make sure that the law latency during redirecting to long urls
  - durability: make sure the short url are durable.
  - uniquiness: make sure that the short url are unique.
